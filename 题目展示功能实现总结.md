# 题目展示功能实现总结

## 🎯 问题分析

### 用户反馈
"为什么前端并没有显示这些题目？"

### 根本原因
前端调用的是 `getTaskQuestions`，该接口查询**数据库中已保存的题目**。但题目只有在用户点"保存"后才会被保存到数据库，在此之前题目只存在于 **Redis 缓存中**。

### 架构问题

```
❌ 错误流程（导致看不到题目）:
┌─────────────────┐
│ 题目生成完成    │
│ 状态: ready     │
└────────┬────────┘
         │
         ↓
┌─────────────────────────────────┐
│ 前端调用 getTaskQuestions       │
│ (查询数据库中已保存的题目)      │
│ SELECT FROM questions           │
└────────┬────────────────────────┘
         │
         ↓
      🔴 结果为空
      (题目还在 Redis 中未保存)
         │
         ↓
    前端看不到题目！

✅ 正确流程（已实现）:
┌─────────────────┐
│ 题目生成完成    │
│ 状态: ready     │
└────────┬────────┘
         │
         ↓
┌──────────────────────────────┐
│ 前端调用 getQuestionPreview  │
│ (从 Redis 读取临时题目)      │
│ REDIS: task:xxx:batch:0      │
└────────┬─────────────────────┘
         │
         ↓
      ✅ 获得题目列表
         │
         ↓
    前端显示题目！
         │
         │ 用户点"保存"
         ↓
   POST /confirm/{id}
   Redis → Database
         │
         ↓
      ✅ 保存成功
```

### 1️⃣ 后端新增 Redis 预览接口

**文件**: `/app/services/exam_service.py`

**新增方法**: `get_questions_from_redis_paginated()`

```python
async def get_questions_from_redis_paginated(
    self,
    task_id: str,
    user_id: int,
    page: int = 1,
    page_size: int = 10
) -> Dict[str, Any]:
    """
    从 Redis 获取临时生成的题目进行预览（分页）
    
    用于题目生成完成后、保存前的预览阶段
    """
```

**核心逻辑**:
- 验证任务所有权
- 从 Redis 调用 `get_all_batches()` 获取所有生成的题目
- 支持分页返回
- 统计题型分布
- 返回与数据库接口相同的格式

**返回格式**:
```json
{
  "total": 8,
  "page": 1,
  "page_size": 10,
  "data": [题目数组],
  "status": "ready",
  "task_type_distribution": {
    "single": 5,
    "multi": 3
  }
}
```

### 3️⃣ 前端新增 API 函数

**文件**: `/src/api/exam.js`

**新增函数**: `getQuestionPreview()`

```javascript
/**
 * 预览临时生成的题目（从 Redis 读取）
 * @param {string} taskId - 任务 ID
 * @param {Object} params - 查询参数
 * @param {number} params.page - 页码，从 1 开始（默认 1）
 * @param {number} params.page_size - 每页数量，最大 50（默认 10）
 * 
 * 说明：
 * - 用于题目生成完成后、保存前的预览阶段
 * - 生成状态为 "ready" 时调用此接口
 * - 返回 Redis 中的临时题目，不包括已保存的数据库题目
 * - 用户确认后调用 confirmSaveQuestions() 保存到数据库
 */
export function getQuestionPreview(taskId, params = {}) {
  return service({
    url: `/exam/questions/preview/${taskId}`,
    method: 'get',
    params: {
      page: params.page || 1,
      page_size: params.page_size || 10
    }
  })
}
```

## 🎓 关键学习点

### 架构设计

**问题**: 为什么需要两个不同的接口？

**答案**: 
- **/preview** - 用于生成阶段，返回 Redis 中的临时题目
- **/task** - 用于查询阶段，返回数据库中已保存的题目
- 两个接口服务于不同的生命周期阶段

### 双轨并行加载

**问题**: 为什么不等进度完成后再加载题目？

**答案**:
- 串行：用户等待 1-2 分钟，黑屏
- 并行：用户 3 秒内看到题目，继续查看，等生成完成

**实现**:
```javascript
// 轨道 A：不停轮询进度
startProgressPolling(taskId)

// 轨道 B：同时加载题目
loadQuestions(taskId)
```

### 分页加载

**问题**: 为什么不一次性加载所有题目？

**答案**:
- 减少内存占用
- 更快的首屏显示
- 更好的用户体验
- 支持大量题目（如 100+ 题）

## 🔍 技术细节

### Redis 存储格式

```
task:{task_id}:batch:0    (字符串) → JSON 序列化的题目列表
task:{task_id}:batch:1    (字符串)
...
task:{task_id}:meta       (哈希)  → 任务元信息
task:{task_id}:question_hashes (集合) → 题目哈希值（去重）
```

### 数据库存储格式

```
questions 表:
- id (主键)
- user_id (用户 ID)
- question_set_name
- content (题目内容)
- type (题型)
- answer
- analysis
- created_at

question_sets 表:
- id (主键)
- user_id (用户 ID)
- name (题集名称)
- created_at
```

### 题型映射

```javascript
// 后端题型 → 前端题型
'choice' → 'single'    // 单选题
'multi' → 'multi'      // 多选题
'judge' → 'judge'      // 判断题
'fill' → 'fill'        // 填空题
'essay' → 'essay'      // 简答题
```

## 📞 故障排查

### 现象 1：前端看不到题目

**可能原因**:
1. ❌ 使用了 `getTaskQuestions()` 而不是 `getQuestionPreview()`
2. ❌ 任务状态不是 `ready`
3. ❌ Redis 中没有题目数据

**解决**:
1. ✅ 确认使用了 `getQuestionPreview()`
2. ✅ 检查后端日志中 status 的值
3. ✅ 检查 Redis 中的批次数据

### 现象 2：题目数量不对

**可能原因**:
1. ❌ 前后端的分页大小不一致
2. ❌ 某个批次没有保存到 Redis
3. ❌ 重复题目被去重

**解决**:
1. ✅ 确保前后端分页大小相同（默认 10）
2. ✅ 检查后端日志中的"已保存"消息
3. ✅ 检查题目去重逻辑

### 现象 3：保存后题目消失

**可能原因**:
1. ❌ 没有调用 `confirmSaveQuestions()`
2. ❌ 数据库连接异常
3. ❌ 题目集关联错误

**解决**:
1. ✅ 确认调用了 confirm API
2. ✅ 检查后端数据库日志
3. ✅ 检查题目集 ID 是否正确
