### 第三步：检查后端接口格式

#### 1. 获取文档列表

**请求：**
```http
GET /api/knowledge/documents
Authorization: Bearer {token}
```

**期望响应格式：**
```json
{
  "code": 200,
  "data": [
    {
      "id": 1,
      "title": "文档标题",
      "file_path": "/path/to/file.pdf",
      "file_type": "pdf",
      "file_size": 102400,
      "chunk_count": 5,
      "status": "processed",
      "created_at": "2026-03-24T10:00:00",
      "updated_at": "2026-03-24T10:00:00"
    }
  ]
}
```

---

#### 2. 获取文档详情

**请求：**
```http
GET /api/knowledge/documents/{id}
Authorization: Bearer {token}
```

**期望响应格式：**
```json
{
  "code": 200,
  "data": {
    "document": {
      "id": 1,

      "content": "文档内容...",

      {
        "id": 1,
        "content": "分块内容...",
        "chunk_index": 0
      }
    ]
  }

```

---

#### 3. 上传文档

**请求：**
```http
POST /api/knowledge/upload
Content-Type: multipart/form-data
Authorization: Bearer {token}

file: {file object}
tags: "1,2,3" (可选)
```

**期望响应格式：**
```json
{
  "code": 200,
  "data": {
    "id": 1,
    "title": "文档标题",
    "file_path": "/path/to/file.pdf",
    "status": "processing"
  },
  "message": "上传成功"
}
```

---

#### 4. 搜索文档

**请求：**
```http
GET /api/knowledge/search?query=关键词&search_type=hybrid&top_k=10
Authorization: Bearer {token}
```

**参数说明：**
- `query`: 搜索关键词
- `search_type`: `keyword` | `vector` | `hybrid`
- `top_k`: 返回结果数量

**期望响应格式：**
```json
{
  "code": 200,
  "data": [
    {
      "doc_id": 1,
      "chunk_id": 10,
      "content": "匹配的内容...",
      "score": 0.95,
      "document": {
        "id": 1,
        "title": "文档标题"
      }
    }
  ]
}
```

---

#### 5. 删除文档

**期望响应格式：**

## 🔧 常见问题排查

### 问题 1：跨域错误 (CORS)

**错误信息：**
```
Access to XMLHttpRequest at 'http://127.0.0.1:8000/api/knowledge/documents' 
from origin 'http://localhost:8080' has been blocked by CORS policy
```

**解决方案：**
在后端添加 CORS 配置（以 FastAPI 为例）：
```python
from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:8080"],  # 前端地址
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

---

### 问题 2：401 未授权

**错误信息：**
```
401 Unauthorized
```

**原因：**
- Token 过期或无效
- 未登录

**解决方案：**
1. 检查是否已登录
2. 检查 localStorage 中是否有 `access_token`
3. 重新登录获取新 token

---

### 问题 3：404 接口不存在

**错误信息：**
```
404 Not Found
```

**检查项：**
1. 后端服务是否启动
2. 接口路径是否正确
3. 请求方法是否正确（GET/POST/DELETE）

---

### 问题 4：500 服务器错误

**错误信息：**
```
500 Internal Server Error
```

**排查步骤：**
1. 查看后端日志
2. 检查数据库连接
3. 检查文件上传目录权限

## 📊 数据格式对照表

### 文档状态 (status)

| 前端期望值 | 说明 |
|-----------|------|
| `processing` | 处理中 |
| `processed` | 已处理 |
| `failed` | 处理失败 |

---

### 文件类型 (file_type)

| 前端期望值 | 说明 |
|-----------|------|
| `pdf` | PDF 文件 |
| `doc` | Word 文档 (.doc) |
| `docx` | Word 文档 (.docx) |
| `txt` | 文本文件 |
| `image` | 图片（用于 OCR） |

---

### 搜索类型 (search_type)

| 前端值 | 后端对应 | 说明 |
|-------|---------|------|
| `keyword` | LIKE 查询 | 关键词匹配 |
| `vector` | 向量相似度 | 语义搜索 |
| `hybrid` | 混合查询 | 两者结合 |

## 🚀 快速开始

1. **启动后端服务**
   ```bash
   cd backend
   python main.py  # 或你的启动命令
   ```

2. **启动前端服务**
   ```bash
   cd frontend
   npm run serve
   ```

3. **访问测试页面**
   ```
   http://localhost:8080/knowledge-api-test
   ```

4. **逐个测试接口**
   - 点击"获取文档列表"测试按钮
   - 查看返回结果
   - 如有错误，查看浏览器控制台和后端日志
