# 题目生成完整流程测试指南

## 🧪 测试场景

### 场景 1：正常生成并预览题目

**步骤**:

1. **打开题目生成页面**
   ```
   前端: QuestionGenerator.vue
   地址: /exam/question-generator
   ```

2. **配置生成参数**
   - 选择知识标签（可选）
   - 选择知识文件（必选，至少 1 个）
   - 选择题型（必选）：如单选题 5 个，多选题 3 个
   - 设置难度：中等
   - 输入题集名称（可选）

3. **点击"生成题目"按钮**
   - 前端发送：`POST /api/exam/questions/generate`
   - 后端返回：`{ "task_id": "xxx", "status": "pending" }`
   - 前端保存 `currentTaskId`

4. **观察生成进度**
   
   **预期行为**:
   - 显示加载动画
   - 显示进度条（0% → 100%）
   - 显示当前步骤信息，如：
     ```
     正在分析文档...
     正在生成第 1/2 批题目（5 题）
     正在生成单选题 batch...
     ...
     ```

5. **观察题目预览**
   
   **预期行为**:
   - 当进度条开始动时，题目开始加载
   - 第一道题目显示在预览卡片中
   - 显示题目内容、选项、答案、解析
   - 显示已生成题目计数：`(已生成 3 题，进度 30%)`

6. **翻页查看题目**
   
   **预期行为**:
   - 点击"下一题"按钮查看下一道题目
   - 显示题号：`第 2/8`
   - 题目内容正确显示
   - 没有数据遗漏

7. **等待生成完成**
   
   **预期行为**:
   - 进度条达到 100%
   - 加载动画消失
   - 所有题目都已加载
   - 显示成功提示：`成功生成 8 道题目`

8. **确认保存题目**
   
   **步骤**:
   - 点击"保存题目"按钮
   - 弹出确认对话框：`确认保存 8 道题目？`
   - 点击"确认"

   **预期行为**:
   - 前端发送：`POST /api/exam/questions/confirm/{task_id}`
   - 后端返回：`{ "message": "保存成功", "saved_count": 8, ... }`
   - 显示成功提示：`成功保存 8 道题目`
   - 页面重置，可以再次生成

### 场景 2：生成失败

**步骤**:

1. **配置无效参数**
   - 不选择任何文件
   - 尝试点击"生成题目"

   **预期行为**:
   - 显示验证错误：`请至少选择一个文件`
   - 生成按钮被禁用

2. **生成超时**
   
   **步骤**:
   - 配置参数正常
   - 生成题目
   - 等待超过 3 分钟（180 秒）

   **预期行为**:
   - 后端停止生成
   - 前端显示超时提示：`生成超时，请稍后重试`
   - 可以取消任务或重新生成

3. **AI 服务异常**
   
   **预期行为**:
   - 前端显示错误提示
   - 显示具体错误信息
   - 提供重新生成选项

---

### 场景 3：网络异常

**步骤**:

1. **生成过程中网络中断**
   
   **预期行为**:
   - 前端尝试继续轮询
   - 显示"正在重试..."提示
   - 网络恢复后继续轮询

2. **加载题目时网络错误**
   
   **预期行为**:
   - 不中断生成流程
   - 继续加载可用的题目
   - 显示部分题目预览

## 🔍 调试技巧

### 1. 查看浏览器控制台日志

```javascript
// QuestionGenerator.vue 中的关键日志
console.log('生成题目响应:', response)           // task_id
console.log('生成进度:', progress)               // status, progress

console.log('原始题目数据:', q)                   // 单个题目

```

### 2. 查看后端日志

```bash
# 监听后端日志
tail -f logs/app.log | grep "✅\|❌"

# 关键日志信息
✅ 批次题目已保存：task:xxx:batch:0 (5 题)    # Redis 保存
✅ essay 完成：3/3                             # 类型完成
✅ 任务状态已更新：status = ready              # 任务就绪
✅ 题目 Hash 已添加：xxx                        # 去重成功
```

### 3. 检查 Redis 中的题目

```bash
# 连接 Redis
redis-cli

# 查看任务的所有批次
> KEYS "task:*:batch:*"

# 查看第一批题目
> GET "task:{task_id}:batch:0"

# 查看题目 Hash 集合（去重）
> SMEMBERS "task:{task_id}:question_hashes"
```

### 4. 检查数据库中的题目

```bash
# 连接数据库
psql -U your_user -d your_db

-- 查询生成的题目
SELECT id, type, content, difficulty, created_at 
FROM questions 
WHERE created_at > NOW() - INTERVAL '1 hour'
ORDER BY created_at DESC;

-- 查询题目集
SELECT * FROM question_sets 
ORDER BY created_at DESC LIMIT 5;
```

### 5. 启用详细日志

在 `app/config.py` 中设置：

```python
# 启用 DEBUG 模式
LOG_LEVEL = "DEBUG"

# 启用 SQL 查询日志（SQLAlchemy）
import logging
logging.getLogger('sqlalchemy.engine').setLevel(logging.INFO)

# 启用 Redis 日志
logging.getLogger('redis').setLevel(logging.DEBUG)
```

## 🎯 性能指标

### 预期性能

| 指标 | 预期值 | 说明 |
|------|--------|------|
| 生成 5 题耗时 | 15-30 秒 | 取决于 AI 速度 |
| 生成 10 题耗时 | 30-60 秒 | 较大批次 |
| 预览接口响应时间 | < 100ms | 从 Redis 读取 |
| 保存接口响应时间 | < 500ms | 写入数据库 |
| 双轨加载延迟 | < 3 秒 | 第一道题显示 |

- 如果生成超过 2 分钟，检查 AI 服务是否正常
- 如果预览接口超过 1 秒，检查 Redis 连接
- 如果保存超过 2 秒，检查数据库连接

## 🚨 常见问题排查

### Q: 为什么看不到题目？

**排查步骤**:
1. 检查 `currentTaskId` 是否正确保存
2. 检查 `getQuestionPreview` 是否被调用（浏览器日志）
3. 检查 API 响应是否包含题目数据
4. 检查 Redis 中是否有批次数据

**解决**:
- 确保使用了新的 `getQuestionPreview` 接口，而不是 `getTaskQuestions`
- 检查任务状态是否为 `ready`

### Q: 为什么题目数量不对？

**排查步骤**:
1. 检查后端日志中的生成日志
2. 检查 Redis 中保存的批次数
3. 检查重复题目的情况

**解决**:
- 如果是重复题目，检查 `_generate_question_hash()` 函数
- 如果是批次分配错误，检查 `execute_batch_generation_task()` 中的类型分配逻辑

### Q: 保存后题目没有出现在题库中？

**排查步骤**:
1. 检查 `confirmSaveQuestions` 是否返回成功
2. 检查数据库中是否有新题目记录
3. 检查题目是否是特定题集的

**解决**:
- 确保调用了 `confirmSaveQuestions` 而不是手动保存
- 检查数据库连接是否正常

### Q: 为什么生成特别慢？

**排查步骤**:
1. 检查 AI 服务是否响应正常
2. 检查网络连接
3. 检查后端日志中的 AI 调用时间

**解决**:
- 减少题目数量测试
- 检查 AI 服务配置（API 密钥、速率限制）
