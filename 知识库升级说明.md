## 二、知识库页面升级

### 1. 搜索功能增强

**文件**: `src/views/knowledge/KnowledgePage.vue`

- **搜索类型选择器**：支持三种搜索模式

</el-select>
```

```

### 2. 自动标签推荐

#### 新增功能：
- **智能推荐标签区**：显示从文档内容自动提取的关键词
- **一键采纳**：可批量添加推荐标签
- **点击添加**：单个点击即可添加为标签

```vue
<!-- 自动标签推荐区 -->
<div class="doc-auto-tags-section" v-if="autoKeywords.length > 0">
  <div class="section-title">
    <span><el-icon><Star /></el-icon> 智能推荐标签</span>
    <el-button type="text" size="small" @click="acceptAutoTags">
      全部采纳
    </el-button>
  </div>
  <div class="doc-tags-list">
    <el-tag
      v-for="(keyword, index) in autoKeywords"
      :key="index"
      type="info"
      effect="plain"
      @click="addAutoTag(keyword)"
    >
      {{ keyword }}
    </el-tag>
  </div>
</div>
```

#### 实现逻辑：
1. 选择文档时自动调用 `loadAutoKeywords(docId)`
2. 后端使用 `jieba.analyse.extract_tags()` 提取关键词
3. 用户可点击添加或批量采纳

## 三、知识图谱可视化页面

### 1. 新建页面组件

**文件**: `src/views/knowledge/KnowledgeGraph.vue`

#### 核心功能：

##### a) 图谱可视化
- 使用 **D3.js** 力导向图展示知识点关系
- 支持缩放、拖拽
- 节点表示知识点，连线表示关系

##### b) 布局切换
```vue
<el-select v-model="graphLayout" @change="updateLayout">
  <el-option label="力导向布局" value="force" />
  <el-option label="环形布局" value="circular" />
  <el-option label="树形布局" value="hierarchical" />
</el-select>
```

##### c) 节点交互
- 点击节点显示详情面板
- 展示节点类型、关联文档数、难度等级等信息

##### d) 导出功能
- 支持导出图谱为 SVG 格式

##### e) 统计信息
- 实时显示节点数、关系数、文档数

#### 技术实现：

```javascript
// 使用 D3.js 力导向模拟
const simulation = d3.forceSimulation(nodes)
  .force('charge', d3.forceManyBody().strength(-300))
  .force('center', d3.forceCenter(width / 2, height / 2))
  .force('link', d3.forceLink(links).id(d => d.id).distance(150))

// 渲染节点
node.append('circle')
  .attr('r', nodeSize.value)
  .attr('fill', d => getNodeColor(d.category))

// 渲染连线
link = svg.append('g')
  .selectAll('line')
  .data(links)
  .join('line')
  .attr('stroke', '#999')
```

## 七、后端接口需求

前端已就绪，需要后端实现以下接口：

### 1. 文档相关
```
GET    /knowledge/documents          # 获取文档列表
GET    /knowledge/documents/{id}     # 获取文档详情
POST   /knowledge/upload             # 上传文档
DELETE /knowledge/documents/{id}     # 删除文档
```

### 2. 搜索相关
```
GET    /knowledge/search?query=xxx&search_type=hybrid&top_k=10
# search_type: keyword | vector | hybrid
```

### 3. 标签相关
```
GET    /knowledge/tags               # 获取标签列表
POST   /knowledge/tags               # 创建标签
DELETE /knowledge/tags/{id}          # 删除标签
POST   /knowledge/documents/{id}/tags        # 添加标签
DELETE /knowledge/documents/{id}/tags/{tagId} # 移除标签
```

### 4. 智能功能
```
GET    /knowledge/documents/{id}/keywords    # 获取关键词
GET    /knowledge/documents/{id}/chunks      # 获取分块
GET    /knowledge/graph                      # 获取知识图谱
POST   /knowledge/documents/{id}/graph       # 生成图谱
```

## 八、数据库表

需要创建以下数据库表（见 `knowledge_db_upgrade.sql`）：

1. **document_chunks** - 文档分块表（核心）
   - 存储分块内容和向量嵌入
   
2. **document_keywords** - 关键词表
   - 存储自动提取的关键词
   
3. **extracted_entities** - 实体表
   - 存储抽取的实体（用于知识图谱）
   
4. **knowledge_points** - 知识点表
   - 知识图谱节点
   
5. **knowledge_relationships** - 知识点关系表
   - 知识图谱边
   
6. **document_knowledge_links** - 文档 - 知识点关联表

## 九、使用流程

### 1. 上传文档
1. 点击"上传文件"按钮

## 十三、注意事项

1. **后端依赖**：
   - 需要安装 text2vec、jieba、PyMuPDF 等 Python 库
   - 需要部署 Qdrant 向量数据库
   - 建议部署 Neo4j 图数据库

2. **性能优化**：
   - 大量文档时建议分页加载
   - 向量搜索建议使用专用数据库
   - 图谱节点过多时考虑聚类展示

3. **数据安全**：
   - 文件上传需要验证类型和大小
   - 敏感信息需要加密存储
   - API 需要 token 认证
