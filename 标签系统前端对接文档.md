# 标签系统前端对接文档

## 系统概述

标签系统采用**用户隔离**设计，每个用户拥有完全独立的标签空间：

1. **智能标签**（AI自动生成）：用户上传文档时自动生成，仅该用户可见
2. **我的标签**（用户自定义）：用户手动创建，仅该用户可见

**核心特性：**
- ✅ 用户A的标签用户B看不到
- ✅ 同名标签在不同用户下互不影响
- ✅ 引用计数为0时自动清理
- ✅ 用户删除账户时，其标签自动级联删除

## 核心功能

### 1. 文档上传/创建时显示标签

**场景**：用户上传文件或粘贴文本后，系统自动生成3个最相关的标签

**API接口**：
```javascript
// 上传文件
POST /api/knowledge/upload
Response: {
  id: 1,
  title: "文档标题",
  message: "上传成功，已自动生成标签：经济学, 社会学, 市场经济"
}

// 创建文本文档
POST /api/knowledge/documents/text
Body: { title: "标题", content: "内容" }
Response: {
  id: 1,
  title: "文档标题",
  message: "创建成功，已自动生成标签：经济学, 社会学, 市场经济"
}
```

**前端展示**：
```
✅ 上传成功！

已自动生成标签：
[🔵 经济学] [🔵 社会学] [🟢 市场经济]

用户可以：
- 点击标签删除
- 点击"编辑"修改标签名称
- 点击"添加标签"手动添加
```

### 3. 按标签检索文档

**场景**：用户选择一个或多个标签进行检索

**API接口**：
```javascript
// 按多个标签检索（OR模式 - 任一标签匹配）
POST /api/knowledge/documents/search-by-tags?tag_ids=1,2,3&match_mode=or&page=1&page_size=20

// 按多个标签检索（AND模式 - 所有标签匹配）
POST /api/knowledge/documents/search-by-tags?tag_ids=1,2,3&match_mode=and&page=1&page_size=20

Response: {
  total: 25,
  documents: [
    {
      id: 1,
      title: "市场经济研究",
      file_type: "pdf",
      status: "completed",
      created_at: "2024-03-28T10:00:00"
      // 注意：知识文档区域不显示标签
    }
  ]
}
```

│ 📄 市场经济研究.pdf              │

### 4. 文档详情中的标签管理

**场景**：查看文档详情时，可以添加或删除标签

**API接口**：
```javascript
// 获取文档的标签
GET /api/knowledge/documents/{doc_id}/tags
Response: {
  document_id: 1,
  tags: [
    { id: 1, name: "经济学", color: "#409EFF" },
    { id: 2, name: "市场经济", color: "#67C23A" }
  ]
}

// 为文档添加标签
POST /api/knowledge/documents/{doc_id}/tags
Body: { tag_ids: [3, 4] }
Response: { success: true }

// 删除文档的标签
DELETE /api/knowledge/documents/{doc_id}/tags/{tag_id}
Response: { success: true }
```

**前端展示**：
```
文档详情

标题：市场经济研究

内容：...

标签：
[🔵 经济学 ×] [🟢 市场经济 ×] [+ 添加标签]

点击 × 删除标签
点击 + 添加标签（弹出标签选择器）
```

## 引用计数机制（后端自动处理）

**前端无需关心，后端自动维护：**

| 操作 | 后端行为 | 前端表现 |
|------|---------|---------|
| 添加标签到文档 | `ref_count + 1` | 标签正常显示 |
| 从文档移除标签 | `ref_count - 1` | 标签正常消失 |
| 标签计数归零 | 自动删除标签 | 该标签从所有列表消失 |

**示例场景：**
```
1. 用户A上传文档 → 自动生成标签 [经济学]
   → knowledge_tags 表: 经济学 (ref_count=1)

2. 用户B上传文档 → 也生成标签 [经济学]
   → knowledge_tags 表: 经济学 (ref_count=2)

3. 用户A删除文档的标签 [经济学]
   → knowledge_tags 表: 经济学 (ref_count=1) ✓ 保留

4. 用户B删除文档的标签 [经济学]
   → knowledge_tags 表: 经济学 (ref_count=0) → 自动删除

5. 之后检索区域不再显示 [经济学] 标签
```

## 用户操作流程

### 上传文档流程
1. 用户选择文件或粘贴文本
2. 点击上传按钮
3. 系统自动生成3个标签
4. 显示生成的标签（可删除/编辑）
5. 用户确认或修改标签

### 检索文档流程
1. 用户在检索区域选择标签
2. 选择匹配模式（任一/所有）
3. 系统返回匹配的文档列表
4. 用户点击文档查看详情

### 管理标签流程
1. 打开文档详情
2. 查看当前标签
3. 点击 × 删除标签
4. 点击 + 添加标签
5. 选择或创建新标签

---

## 注意事项

1. **标签数量限制**：AI自动生成最多3个标签
2. **标签类型区分**：智能标签和我的标签分开显示
3. **检索逻辑**：
   - OR模式：文档包含任一选中标签即匹配
   - AND模式：文档必须包含所有选中标签
4. **标签颜色**：前端根据颜色编码显示不同类型的标签
5. **权限控制**：用户只能管理自己文档的标签

## 错误处理

### 常见错误
- 标签名称已存在：提示用户更换名称
- 标签不存在：提示用户刷新页面
- 网络错误：提示用户稍后重试

### 错误响应格式

  detail: "错误信息描述"
}
